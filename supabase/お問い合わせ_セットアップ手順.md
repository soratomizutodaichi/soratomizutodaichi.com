# お問い合わせ機能（フェーズ1）セットアップ手順

対象サイト：soratomizutodaichi.com
Supabase プロジェクト：`gcmtnzvcoovjqxanbmqc`

---

## 全体像
- **フォーム／DB保存／管理画面**：手順1〜3で有効化（メール設定不要で動きます）
- **新着メール通知（あなた宛）**：手順4（Resend の設定が必要）

デプロイ（GitHubへ push）は、**手順1（SQL実行）を終えてから**行ってください。テーブルが無い状態で公開すると送信がエラーになります。

---

## 手順1：テーブルとRLSを作成（必須）
1. Supabase ダッシュボード → 左メニュー **SQL Editor**
2. `supabase/contacts_setup.sql` の内容を貼り付けて **Run**
3. エラーが出なければ完了（`contacts` テーブルとポリシーが作成されます）

## 手順2：公開サインアップを無効化（必須・重要）
個人情報を守るための必須設定です。
1. Authentication → **Providers → Email**
2. **「Allow new users to sign up」を OFF**
   → これで「ログイン済みユーザー＝あなたが作った管理者だけ」になります。

## 手順3：管理者ユーザーを用意（閲覧用）
- Authentication → **Users → Add user** で管理者のメール／パスワードを作成
  （既に管理画面にログインできているなら不要）
- 管理画面：`https://soratomizutodaichi.com/admin/contacts.html`
  （`admin/login.html` からログイン後に閲覧できます）

## ここまでで動作確認
- サイトの「お問い合わせ」から試しに送信 → 完了メッセージが出る
- 管理画面 `admin/contacts.html` に届いた内容が表示される
- ダッシュボードに「未対応 ◯件」が表示される

## 手順4：新着メール通知を有効化（任意）

### 4-1. Resend アカウント（あなたの作業）
1. https://resend.com でアカウント作成（無料枠：月3,000通・1日100通の想定）
2. **Domains** で `soratomizutodaichi.com` を追加し、表示される DNS レコードを設定してドメイン認証
3. **API Keys** で API キーを1つ発行（`re_...`）
   - このキーは秘密情報です。**GitHub など公開場所には絶対に置かないでください。**

### 4-2. Edge Function をデプロイ
- Supabase ダッシュボード → **Edge Functions → Deploy a new function**
- 名前：`notify-contact`
- コード：`supabase/functions/notify-contact/index.ts` の内容を貼り付けてデプロイ

### 4-3. シークレット（環境変数）を設定
Edge Functions → notify-contact → **Secrets** に以下を登録：

| 名前 | 値 |
|------|----|
| `RESEND_API_KEY` | Resend の API キー |
| `CONTACT_NOTIFY_TO` | 通知を受け取る管理者メール |
| `CONTACT_NOTIFY_FROM` | 例：`no-reply@soratomizutodaichi.com`（認証済みドメインのアドレス） |
| `WEBHOOK_SECRET` | 任意の長いランダム文字列（なりすまし防止の合言葉） |

### 4-4. Database Webhook を作成
- Database → **Webhooks → Create a new hook**
- Table：`contacts`／Events：**Insert**
- Type：**Supabase Edge Functions** → `notify-contact`
- HTTP Headers に `x-webhook-secret: （4-3のWEBHOOK_SECRETと同じ値）` を追加

### 4-5. テスト
- サイトから試しに1件送信 → 管理者メールに通知が届けば完了

## セキュリティ設計メモ
- `anon` キーは **INSERT のみ**（送信専用）。閲覧・更新・削除は不可。
- 閲覧・対応更新は **ログイン管理者のみ**（RLSで制御）。
- 入力値は**アプリ側とDBの CHECK 制約で二重に検証**。
- スパム対策は**ハニーポット＋送信時間チェック**（CAPTCHAなし）。
- Resend の API キーは **Edge Function のシークレットのみ**に保存し、リポジトリには置かない。
- この `supabase/` フォルダ内のファイルには秘密情報は含まれません（スキーマとロジックのみ）。
