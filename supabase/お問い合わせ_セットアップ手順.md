## 全体像
- **フォーム／DB保存／管理画面での閲覧**：手順1〜3（メール設定なしで動きます）

デプロイ（GitHubへ push）は、**手順1（SQL実行）を終えてから**行ってください。

## 手順1：テーブルとRLSを作成（必須）
Supabase ダッシュボード → **SQL Editor** で、次の2ファイルを順に貼り付けて **Run**：
1. `supabase/contacts_setup.sql`（お問い合わせ本体）
2. `supabase/contact_replies_setup.sql`（返信履歴）

いずれも「Success. No rows returned」が正常です。

## 手順2：公開サインアップを無効化（必須・重要）
Authentication → Providers → Email →「**Allow new users to sign up**」を **OFF**。
→ 「ログイン済みユーザー＝あなたが作った管理者だけ」になり、個人情報を守れます。

## 手順3：管理者ユーザーを用意
- Authentication → **Users → Add user** で管理者のメール／パスワードを作成
- 管理画面：`admin/login.html` → ログイン後 `admin/contacts.html`

### ここまでで動作確認
- サイトの「お問い合わせ」から送信 → 完了メッセージ
- `admin/contacts.html` に届いた内容が表示される
- ダッシュボードに「未対応 ◯件」が表示される

## 手順4：メール（通知＋返信）を有効化

### 4-1. Resend アカウント（あなたの作業）
1. https://resend.com でアカウント作成（無料枠：月3,000通・1日100通の想定・**追加料金なし**）
2. **Domains** で `soratomizutodaichi.com` を追加し、表示DNSレコードを設定して認証
3. **API Keys** で API キー（`re_...`）を発行
   - 秘密情報です。**公開リポジトリ等には絶対に置かないでください。**

### 4-2. Edge Function を2つデプロイ
Supabase ダッシュボード → **Edge Functions → Deploy a new function**

| 関数名 | 元コード | JWT検証(Verify JWT) |
|--------|----------|----------------------|
| `notify-contact` | `supabase/functions/notify-contact/index.ts` | **OFF**（Webhookから呼ぶ＋合言葉で保護） |
| `send-contact-reply` | `supabase/functions/send-contact-reply/index.ts` | **OFF**（関数内で管理者トークンを検証） |

※ Verify JWT は各関数の設定（Details → Function Configuration）で切り替えます。
※ `send-contact-reply` を OFF にする理由：ブラウザのCORSプリフライト（OPTIONS）は
　 トークンを持たないため、Verify JWT を ON にすると弾かれて送信が失敗します。
　 代わりに**関数コード内で毎回トークンを検証**し、匿名・未ログインを拒否しています
　 （安全性は担保されます）。

### 4-3. シークレット（環境変数）を設定
Edge Functions → **Secrets**（プロジェクト共通）に登録：

| 名前 | 値 | 用途 |
|------|----|------|
| `RESEND_API_KEY` | Resend の API キー | 通知・返信の両方 |
| `CONTACT_NOTIFY_FROM` | 例：`no-reply@soratomizutodaichi.com`（認証済みドメイン） | 送信元 |
| `CONTACT_NOTIFY_TO` | 通知を受け取る**指定アドレス**（あなたの受信箱） | 通知先＋返信のReply-To |
| `WEBHOOK_SECRET` | 任意の長いランダム文字列 | 通知の改ざん防止（notify-contact用） |
| `ALLOWED_ORIGIN` | `https://soratomizutodaichi.com` | 返信送信の許可元（省略可・既定同値） |
| `ADMIN_EMAILS` | （任意）送信を許可する管理者メール（カンマ区切り） | 返信送信の追加制限 |

`SUPABASE_URL` / `SUPABASE_ANON_KEY` / `SUPABASE_SERVICE_ROLE_KEY` は自動で入るため設定不要です。

### 4-4. 新着通知の Database Webhook を作成
Database → **Webhooks → Create a new hook**
- Table：`contacts`／Events：**Insert**
- Type：**Supabase Edge Functions** → `notify-contact`
- HTTP Headers に `x-webhook-secret: （WEBHOOK_SECRETと同じ値）` を追加

### 4-5. テスト
1. サイトから1件送信 → **指定アドレスに通知メール**が届く
2. `admin/contacts.html` で該当を開く → 返信欄に入力 → **「✉ この内容でメール送信」**
3. お客様アドレスに返信メールが届く／履歴に記録され「対応済み」になる
4. お客様が返信すると、`CONTACT_NOTIFY_TO`（あなたの受信箱）に届く

## セキュリティ設計メモ
- `anon` キーは **INSERT のみ**（送信専用）。閲覧・更新・削除は不可。
- 閲覧・返信は **ログイン管理者のみ**（RLS＋Edge Functionのトークン検証）。
- 返信の送信先は**保存済みの宛先だけ**。任意アドレスへは送れない（踏み台化防止）。
- `contact_replies` への書き込みは **service role（Edge Function内）のみ**。
- 入力値はアプリ側とDBの CHECK 制約で二重検証。スパム対策はハニーポット＋送信時間チェック。
- Resend／サービスキーは **Edge Function のシークレットのみ**。リポジトリ内ファイルに秘密情報は含みません。
